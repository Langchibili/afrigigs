public async rankBids(jobContext, bidList) {
  const isGlobal = jobContext.scope === "FOR_EVERYONE";
  
  return bidList.sort((a, b) => {
    if (!isGlobal) {
      // Sort by proximity match first if restricted regionally
      const proximityA = a.user.native_last_city === jobContext.target_city ? 0 : 1;
      const proximityB = b.user.native_last_city === jobContext.target_city ? 0 : 1;
      if (proximityA !== proximityB) return proximityA - proximityB;
    }
    // Secondary sort fallbacks
    if (b.user.trust_score_yes !== a.user.trust_score_yes) {
      return b.user.trust_score_yes - a.user.trust_score_yes;
    }
    return b.user.profile_completion_skilled - a.user.profile_completion_skilled;
  });
}

# AfriGigs — Full Project Plan

**A localized, pan-African, mobile-first gig marketplace**

---

## 1. Executive Summary

AfriGigs connects independent workers ("gig-runners") with local and international job posters across African markets, using an escrow-backed payment system, dynamic per-country bidding rules, and privacy-safe proximity matching. This plan expands the original blueprint into a build-ready roadmap: architecture, design system, data model, workflows, team structure, timeline, and risk plan.

**Tech stack (confirmed):**
- Frontend: **Next.js** + **MUI (Material UI)** as the component library, **Framer Motion** for animation, **custom CSS Grid/Flexbox layout system** (MUI's `Grid`/`Grid2` is explicitly *not* used — see §4.3)
- Mobile shell: **React Native** WebView wrapper (native GPS + push notifications bridged via `postMessage`)
- Backend: **Strapi** (headless CMS as business-logic/API layer)
- Database: **MySQL**
- Payments: Paystack / Flutterwave / DPO Group aggregators
- Geocoding: Google Geocoding API (reverse-geocode only, never raw coordinates to other users)

## 4. Frontend Design System

### 4.1 Visual Direction
"Clean, modern, trustworthy" — the dark Pan-African palette from the blueprint is kept but refined for MUI theming and layered depth (glassy elevation, soft glow accents) rather than flat blocks.

**Palette (as MUI theme tokens):**
```js
palette: {
  mode: 'dark',
  primary:   { main: '#00BFA6', dark: '#009688', light: '#5BE0CB' }, // Unity Green, brightened for AA contrast
  secondary: { main: '#FFC107', dark: '#C79100', light: '#FFD54F' }, // Gold
  error:     { main: '#EF5350', dark: '#D32F2F' },                   // Alert Red
  background:{ default: '#121212', paper: '#1A1A1A' },
  divider:   'rgba(255,255,255,0.08)',
}
```

**Elevation strategy:** replace MUI's default flat `elevation` shadows with a custom shadow scale (soft, colored, low-spread) — e.g. cards get `box-shadow: 0 8px 24px rgba(0,0,0,0.45), 0 0 0 1px rgba(255,255,255,0.04)`, active/hovered cards add a faint primary-color glow (`0 0 20px rgba(0,191,166,0.15)`).

**Gradients:** used sparingly as accents, not backgrounds — CTA buttons use a 135° green→teal gradient; the "Verified" badge uses a gold gradient sheen; hero section uses a radial gradient vignette behind a mesh/grain texture at low opacity.

**Typography:** `Inter` or `General Sans` for UI, tighter tracking on headings, a numeric tabular-figure font (e.g. `Space Grotesk` or the same family's tabular variant) for wallet balances and bid amounts so numbers don't jitter on update.

### 4.2 Motion (Framer Motion)
- **Page transitions:** shared-layout fade/slide (`AnimatePresence` + `layoutId`) between `/gigs` list and `/gigs/[id]` detail — the gig card animates into the detail hero.
- **Micro-interactions:** button press = scale 0.97 + shadow compress; bid submission = success checkmark draws in via `pathLength` animation.
- **Skeleton → content:** staggered fade/slide-up (`staggerChildren`) as gig cards populate, avoiding layout pop.
- **Verification badge:** subtle looping shimmer (gradient position animation) on "Unverified" state to draw the eye without being obnoxious; stops and morphs to a solid check once verified.
- **Trust/Proximity sort changes:** cards use `layout` prop so re-sorting animates positions rather than jump-cutting.
- Respect `prefers-reduced-motion` — all animations wrapped with a reduced-motion fallback (opacity-only).

### 4.3 Custom Grid System (No MUI Grid)
Per requirement, MUI's `Grid`/`Grid2` is not used anywhere. Instead, a small custom layout primitive set is built once and reused:

```
components/layout/
  Stack.tsx      → thin wrapper over CSS flexbox (gap, direction, align, justify, wrap)
  GridBox.tsx    → thin wrapper over CSS Grid (columns, rows, gap, responsive column-count via props + MUI useMediaQuery)
  Container.tsx  → max-width + responsive padding shell
```

Example `GridBox` usage for the gig list:
```tsx
<GridBox columns={{ xs: 1, sm: 2, md: 3, lg: 4 }} gap={3}>
  {gigs.map(g => <GigCard key={g.id} {...g} />)}
</GridBox>
```
Internally this resolves to native `display: grid; grid-template-columns: repeat(n, 1fr); gap: ...` with breakpoints pulled from the MUI theme's `theme.breakpoints`, so spacing/typography stay token-consistent with MUI while layout math stays custom and fully controllable (e.g. asymmetric featured-gig layouts, masonry-style profile galleries) which MUI's Grid can't easily do.

### 4.4 Core Component Inventory
- `GigCard` (compact/expanded variants), `TrustBadge`, `VerifiedTag`, `BidTicket`, `WalletBalanceChip`, `ProfileCompletionRing` (animated SVG progress), `CountryCurrencySelector`, `OnboardingStepper` (custom, not MUI `Stepper` visuals but same a11y pattern), `EscrowStatusTimeline`.

## 6. Core Workflows

### 6.1 Onboarding & Verification
`Register → Basic details + ID number + skills category → Instantly browse & bid (Unverified badge)` → optional `Upload scripted video selfie → Strapi review queue → Verified badge`.
- Unverified users can bid but their `BidTicket` renders with a caution chip; some employers may filter to verified-only.
- Verification review can start manual (admin panel in Strapi) and later add automated liveness/ID-match checks.

### 6.2 GPS & Proximity Matching
- Native app reads device GPS → sends raw lat/lng to WebView via a JS bridge message.
- Server reverse-geocodes to city-level only (`native_last_city`) — **raw coordinates are never persisted or exposed to other users**, only the resolved city string.
- `rankBids()` sorts: proximity (if job scope is `LOCAL`) → `trust_score_yes` → `profile_completion_skilled`, exactly as in the blueprint's TypeScript sort function — this ships as a Strapi custom controller.

### 6.3 Bidding Thresholds
- International bid (poster country ≠ bidder country): minimum **$5.00 USD** fixed floor.
- Domestic bid: minimum = `country.currency.minimum_national_bid`.
- Enforced in a Strapi lifecycle hook (`beforeCreate` on Bid) — errors surface as inline form validation in the Next.js `BidTicket` component, not raw 500s.

### 6.4 Escrow Ledger
`Poster funds job (Paystack/Flutterwave/DPO) → funds held in EscrowTransaction → worker delivers → poster confirms completion → funds move to Wallet.balance → worker withdraws to mobile money`.
- Add a **dispute path**: if poster doesn't confirm within N days of delivery marked, job auto-flags for admin review rather than silently staying locked.
- Add **partial release** support for milestone-based jobs (future phase).

## 10. Risks & Mitigations

| Risk | Mitigation |
|---|---|
| Currency/FX mismatches on international bids | Lock `proposed_amount_usd` at bid time using a snapshot rate; don't recompute live |
| Fake/GPS-spoofed verification videos | Manual review queue at launch; plan for liveness-detection vendor later |
| Escrow disputes with no resolution path | Build the auto-flag-to-admin timeout from day one, not as an afterthought |
| MUI + custom grid drift (inconsistent spacing) | Pull all spacing from `theme.spacing()`, never hardcode px, enforced via lint rule |
| Low-end Android performance with heavy animation | Gate non-essential Framer Motion effects behind a device-capability/`prefers-reduced-motion` check |

A.6 — Custom Grid / Layout System (MUI Grid is NOT used)
--------------------------------------------------------------------------------
Build once, reuse everywhere:

    Thin wrapper over CSS flexbox.

    (e.g. direction={{ xs: 'column', md: 'row' }}).

    Thin wrapper over native CSS Grid.

    Props: maxWidth ('sm'|'md'|'lg'|'xl'|'full'), disableGutters.

D.3 — Custom Endpoints
--------------------------------------------------------------------------------

POST /api/users/me/location
  Auth: required
  Body: { latitude: number, longitude: number }
  Behavior: reverse-geocodes server-side (Google Geocoding API) and stores
  ONLY the resolved city on user.native_last_city. Raw coordinates are
  never persisted or returned in the response — this is a deliberate
  privacy design constraint from the original blueprint (GPS privacy /
  city-level only). The RN wrapper calls this after reading device GPS;
  the Next.js web app never needs to call it directly.
  Response: { success: true, native_last_city: string }

GET /api/jobs/:id/rank-bids
  Auth: required (job owner typically)
  Behavior: returns the job's bids sorted by proximity to target_city
  (skipped if job.scope === 'FOR_EVERYONE') → trust_score_yes descending →
  profile_completion_skilled descending. This is the algorithm from the
  original blueprint's rankBids() TypeScript sketch, now a real endpoint.
  Response: { success: true, data: Bid[] }  (each Bid populated with user)

POST /api/escrow-transactions/:id/release
  Auth: required, must be the job's owner
  Behavior: moves the held/funded amount into the assigned worker's
  Wallet.balance, marks the transaction 'released', and the job
  'completed' with escrow_status 'released' + completed_at stamped.
  Response: { success: true, data: { released: true, walletId, amount } }

D.4 — Business Rules Enforced Server-Side (lifecycles — not just UI validation)
--------------------------------------------------------------------------------
These live in src/api/<type>/content-types/<type>/lifecycles.ts. The
frontend MUST treat their error responses as authoritative — never assume
a client-side check alone is sufficient, always surface the server's
rejection message inline (e.g. in BidTicket).

  Bid.beforeCreate
    - Rejects bids on a job that isn't status='open'
    - Rejects self-bids (user === job.owner)
    - International bid (job.owner.country !== bidder.country): must be
      >= $5.00 USD (proposed_amount_usd)
    - Domestic bid (same country): must be >= job.country.currency.
      minimum_national_bid (proposed_amount_local)
    - Error messages are human-readable and safe to show directly in the
      BidTicket form, e.g.: "Bid rejected. The domestic floor limit for
      this area is ZK50."

  Bid.beforeUpdate
    - proposed_amount_local / proposed_amount_usd cannot be changed after
      creation. Frontend flow: "Withdraw" (status → withdrawn) then submit
      a new bid, rather than an edit form.

  Review.afterCreate
    - rating >= 4 increments to_user.trust_score_yes by 1. Frontend should
      optimistically NOT update trust score locally — refetch the profile
      after a review is submitted to reflect the true server value.

  VerificationSubmission.afterUpdate
    - status → 'approved' sets user.is_verified_human = true and stamps
      reviewed_at. This happens when an admin acts in the Strapi admin
      panel; frontend should poll GET /api/verification-submissions
      (filtered to the current user) on the /verify page to detect the
      change, or refetch /api/users/me on next load.

  Hourly cron (config/cron-tasks.ts)
    - Any Job with status='in_progress' AND escrow_status='held' AND
      completion_confirmation_deadline < now gets auto-flagged to
      status='disputed'. Frontend should render DisputeFlagBanner
      wherever job.status === 'disputed' and surface a "Contact support"
      or admin-review messaging path — there is currently no in-app
      dispute-resolution page, this is handled in the admin panel.

D.5 — Permissions Baseline
--------------------------------------------------------------------------------
Set via src/index.ts bootstrap(): the `public` role has read-only access
(find/findOne) to Currency, Country, and SkillsCatalog only — needed so
Landing and Onboarding can populate dropdowns before login. Everything else
(Job, Bid, Wallet, EscrowTransaction, profiles, reviews, messages) is
restricted to `authenticated` by default; owner-scoped write restrictions
(e.g. "only the job owner can update their Job") should be configured as
Strapi policies in the admin panel's Roles UI — the frontend should not
assume any write is publicly permitted beyond what's listed above, and
should handle 403 Forbidden responses gracefully (e.g. hide "Edit" buttons
for non-owners, but always also handle the server rejection as the source
of truth).

D.6 — Payment Gateway Integration Status
--------------------------------------------------------------------------------
Gateway webhook receivers (Paystack / Flutterwave / DPO) are NOT yet wired
to a live route — this is the one open backend seam pending live gateway
credentials. When built, expect: POST /api/webhooks/:gateway, which
verifies the gateway's signature, creates an EscrowTransaction with
status='funded', and updates the linked Job.escrow_status. The frontend's
"Post a Job" funding step and "Wallet" withdrawal step should be built
against this expected shape now, with the actual checkout redirect/SDK
call per gateway (Paystack Inline, Flutterwave Standard, DPO redirect)
wired in once the webhook exists.

// src/api/escrow-transaction/controllers/escrow-transaction.ts

import { factories } from '@strapi/strapi';

export default factories.createCoreController('api::escrow-transaction.escrow-transaction', ({ strapi }) => ({
    /**
     * Job-poster-only. Moves held escrow into the assigned worker's Wallet
     * and closes out the job.
     */
    async release(ctx) {
        try {
            const { id } = ctx.params;
            const requester = ctx.state.user;

            const transaction: any = await strapi.db.query('api::escrow-transaction.escrow-transaction').findOne({
                where: { id },
                populate: { job: { populate: ['owner', 'assigned_worker'] }, currency: true },
            });

            if (!transaction) return ctx.notFound('Escrow transaction not found.');
            if (transaction.status !== 'held' && transaction.status !== 'funded') {
                return ctx.badRequest(`Transaction is already ${transaction.status}.`);
            }

            const job = transaction.job;
            if (!job || !job.owner) return ctx.badRequest('Escrow transaction is not linked to a valid job.');
            if (requester && job.owner.id !== requester.id) {
                return ctx.forbidden('Only the job poster can release escrow funds.');
            }
            if (!job.assigned_worker) return ctx.badRequest('Job has no assigned worker to release funds to.');

            const workerId = job.assigned_worker.id;

            let wallet: any = await strapi.db.query('api::wallet.wallet').findOne({
                where: { user: workerId },
            });

            if (!wallet) {
                wallet = await strapi.db.query('api::wallet.wallet').create({
                    data: { user: workerId, balance: 0, pending_balance: 0, currency: transaction.currency?.id },
                });
            }

            await strapi.db.query('api::wallet.wallet').update({
                where: { id: wallet.id },
                data: { balance: Number(wallet.balance) + Number(transaction.amount) },
            });

            await strapi.db.query('api::escrow-transaction.escrow-transaction').update({
                where: { id },
                data: { status: 'released' },
            });

            await strapi.db.query('api::job.job').update({
                where: { id: job.id },
                data: { status: 'completed', escrow_status: 'released', completed_at: new Date() },
            });

            ctx.send({
                success: true,
                data: { released: true, walletId: wallet.id, amount: transaction.amount },
            });
        } catch (error) {
            strapi.log.error('Error releasing escrow:', error);
            ctx.internalServerError('Failed to release escrow');
        }
    },
}));

// src/api/bid/content-types/bid/lifecycles.ts

const FIXED_USD_MIN = 5.0;

function extractId(relation: any): number | null {
    if (relation === null || relation === undefined) return null;
    if (typeof relation === 'number') return relation;
    if (typeof relation === 'string') return Number(relation);
    if (relation.id) return relation.id;
    if (Array.isArray(relation.connect) && relation.connect[0]) {
        return relation.connect[0].id ?? relation.connect[0];
    }
    return null;
}

export default {
    /**
     * Enforces the national vs. international bid floor:
     *   - International (job owner country !== bidder country): min $5.00 USD
     *   - Domestic: min = country.currency.minimum_national_bid (local currency)
     * Also blocks bids on closed jobs and self-bids.
     */
    async beforeCreate(event: any) {
        const { data } = event.params;

        const jobId = extractId(data.job);
        const userId = extractId(data.user);

        if (!jobId || !userId) {
            throw new Error('A bid requires both a job and a user.');
        }

        const job: any = await strapi.db.query('api::job.job').findOne({
            where: { id: jobId },
            populate: { owner: { populate: ['country'] }, country: { populate: ['currency'] } },
        });

        if (!job) throw new Error('Cannot bid on a job that does not exist.');
        if (job.status !== 'open') throw new Error('This job is no longer accepting bids.');

        const bidder: any = await strapi.db.query('plugin::users-permissions.user').findOne({
            where: { id: userId },
            populate: ['country'],
        });

        if (!bidder) throw new Error('Bidder account could not be resolved.');
        if (job.owner && job.owner.id === bidder.id) {
            throw new Error('You cannot bid on your own job.');
        }

        const ownerCountryId = job.owner?.country?.id ?? null;
        const bidderCountryId = bidder.country?.id ?? null;
        const isInternational = ownerCountryId !== bidderCountryId;

        if (isInternational) {
            const proposedUsd = Number(data.proposed_amount_usd);
            if (!proposedUsd || proposedUsd < FIXED_USD_MIN) {
                throw new Error(
                    `International bids must meet the minimum fallback rate of $${FIXED_USD_MIN.toFixed(2)} USD.`
                );
            }
        } else {
            const domesticMin = job.country?.currency ? Number(job.country.currency.minimum_national_bid) : 0;
            const proposedLocal = Number(data.proposed_amount_local);
            const symbol = job.country?.currency ? job.country.currency.symbol : '';

            if (!proposedLocal || proposedLocal < domesticMin) {
                throw new Error(`Bid rejected. The domestic floor limit for this area is ${symbol}${domesticMin}.`);
            }
        }
    },

    /**
     * Bid amounts are immutable once submitted — withdraw and re-bid instead
     * of editing, so the floor-check above can't be bypassed via update.
     */
    async beforeUpdate(event: any) {
        const { data } = event.params;
        if (data && (data.proposed_amount_local !== undefined || data.proposed_amount_usd !== undefined)) {
            throw new Error('Bid amounts cannot be edited after submission. Withdraw and re-bid instead.');
        }
    },
};

// src/api/job/controllers/job.ts

import { factories } from '@strapi/strapi';

export default factories.createCoreController('api::job.job', ({ strapi }) => ({
    /**
     * Returns a job's bids sorted by:
     *   1. Proximity to target_city (skipped when scope === 'FOR_EVERYONE')
     *   2. trust_score_yes (descending)
     *   3. profile_completion_skilled (descending)
     */
    async rankBids(ctx) {
        try {
            const { id } = ctx.params;

            const job: any = await strapi.db.query('api::job.job').findOne({
                where: { id },
                populate: { bids: { populate: ['user'] } },
            });

            if (!job) return ctx.notFound('Job not found');

            const isGlobal = job.scope === 'FOR_EVERYONE';

            const ranked = [...(job.bids || [])].sort((a: any, b: any) => {
                if (!isGlobal) {
                    const proximityA = a.user?.native_last_city === job.target_city ? 0 : 1;
                    const proximityB = b.user?.native_last_city === job.target_city ? 0 : 1;
                    if (proximityA !== proximityB) return proximityA - proximityB;
                }

                const trustA = a.user?.trust_score_yes || 0;
                const trustB = b.user?.trust_score_yes || 0;
                if (trustB !== trustA) return trustB - trustA;

                const completionA = a.user?.profile_completion_skilled || 0;
                const completionB = b.user?.profile_completion_skilled || 0;
                return completionB - completionA;
            });

            ctx.send({ success: true, data: ranked });
        } catch (error) {
            strapi.log.error('Error ranking bids:', error);
            ctx.internalServerError('Failed to rank bids');
        }
    },
}));

export default {
    /**
     * A 4-5 star review increments the recipient's trust_score_yes, which
     * feeds directly into the job.rankBids proximity/trust sort algorithm.
     */
    async afterCreate(event: any) {
        const { result } = event;

        const rating = result?.rating;
        if (!rating || rating < 4) return;

        const review: any = await strapi.db.query('api::review.review').findOne({
            where: { id: result.id },

/**
 * Get normalized relative path from root directory
 */
function getRelativePath(filePath) {
    const relative = path.relative(process.cwd(), filePath);
    return relative ? relative.replace(/\\/g, '/') : '.';
}

/**
 * Check if a directory should be ignored
 */
function shouldIgnoreDirectory(dirPath) {
    const relativePath = getRelativePath(dirPath);
    const dirName = path.basename(dirPath);

    // Check by directory name (global)
    if (IGNORED_DIRECTORIES.includes(dirName)) {
        return true;
    }

    // Check exact directory path
    if (IGNORED_DIRECTORY_PATHS.includes(relativePath)) {
        return true;
    }

    return false;
}

/**
 * Check if a file should be ignored
 */
function shouldIgnoreFile(fileName, filePath) {
    const relativePath = getRelativePath(filePath);
    const ext = path.extname(fileName).toLowerCase();

    // Check by filename
    if (ALL_IGNORED_FILES.includes(fileName)) {
        return true;
    }

    // Check by extension
    if (IGNORED_FILE_EXTENSIONS.includes(ext)) {
        return true;
    }

    // Check exact file path
    if (ALL_IGNORED_FILE_PATHS.includes(relativePath)) {
        return true;
    }

    // Check if file is in an ignored directory
    const fileDir = path.dirname(relativePath);
    if (IGNORED_DIRECTORY_PATHS.includes(fileDir)) {
        return true;
    }

    // Check hidden files (except allowed ones)
    if (fileName.startsWith('.') && !fileName.startsWith('.env')) {
        const allowedHidden = ['.eslintrc', '.prettierrc', '.babelrc', '.npmrc'];
        if (!allowedHidden.some(allowed => fileName.startsWith(allowed))) {
            return true;
        }
    }

    return false;
}

/**
 * Recursively traverse directory and collect all non-ignored files
 */
async function traverseDirectory(dirPath) {
    const files = [];

    // Skip if this directory should be ignored
    if (shouldIgnoreDirectory(dirPath)) {
        console.log(`  ⏭️  Skipping directory: ${getRelativePath(dirPath)}`);
        stats.skipped++;
        return files;
    }

    try {
        const items = await readdir(dirPath);

        for (const item of items) {
            const fullPath = path.join(dirPath, item);

            try {
                const fileStat = await stat(fullPath);

                if (fileStat.isDirectory()) {
                    const subFiles = await traverseDirectory(fullPath);
                    files.push(...subFiles);
                } else if (fileStat.isFile()) {
                    if (!shouldIgnoreFile(item, fullPath)) {
                        files.push(fullPath);
                    } else {
                        console.log(`  ⏭️  Skipping file: ${getRelativePath(fullPath)}`);
                        stats.skipped++;
                    }
                }
            } catch (err) {
                console.error(`  ❌ Error accessing ${fullPath}: ${err.message}`);
                stats.errors++;
            }
        }
    } catch (err) {
        console.error(`  ❌ Error reading directory ${dirPath}: ${err.message}`);
        stats.errors++;
    }

    return files;
}

/**
 * Read file content with encoding detection
 */
async function readFileContent(filePath) {
    try {
        const buffer = await readFile(filePath);

        // Try UTF-8 first
        try {
            return buffer.toString('utf8');
        } catch (err) {
            // Fallback to latin1 for binary files
            return `[BINARY FILE]\n${buffer.toString('base64')}\n[/BINARY FILE]`;
        }
    } catch (err) {
        console.error(`  ❌ Error reading file: ${err.message}`);
        stats.errors++;
        return `[ERROR: ${err.message}]`;
    }
}

 * Format file size for display

function formatFileSize(bytes) {
    if (bytes < 1024) return `${bytes} bytes`;
    if (bytes < 1024 * 1024) return `${(bytes / 1024).toFixed(2)} KB`;
    return `${(bytes / (1024 * 1024)).toFixed(2)} MB`;
}

/**
 * Create the complete source code document
 */
async function createCodeDocument() {
    console.log('\n📂 Scanning directory:', process.cwd());
    console.log('🔍 Gathering files...\n');

    const allFiles = await traverseDirectory(process.cwd());

    if (allFiles.length === 0) {
        console.log('❌ No files found to process!');
        return null;
    }

    console.log(`\n📊 Found ${allFiles.length} files to process\n`);

    let outputContent = '';

    // Process each file
    for (let i = 0; i < allFiles.length; i++) {
        const filePath = allFiles[i];
        const relativePath = getRelativePath(filePath);

        console.log(`📝 Processing (${i + 1}/${allFiles.length}): ${relativePath}`);

        // Add file header with path
        outputContent += `\n${'='.repeat(80)}\n`;
        outputContent += `FILE: ${relativePath}\n`;
        outputContent += `${'='.repeat(80)}\n\n`;

        // Add file content
        const content = await readFileContent(filePath);
        outputContent += content;

        // Ensure trailing newline
        if (content && !content.endsWith('\n')) {
            outputContent += '\n';
        }

        stats.processed++;
    }

    // Write to file
    try {
        await writeFile(OUTPUT_FILE, outputContent, 'utf8');
        console.log(`\n✅ Created: ${OUTPUT_FILE}`);
        console.log(`📏 Size: ${formatFileSize(outputContent.length)}`);
        return outputContent;
    } catch (err) {
        console.error(`❌ Error writing file: ${err.message}`);
        return null;
    }
}

}

// src/extensions/users-permissions/strapi-server.ts

/**
 * Adds POST /api/users/me/location. The RN wrapper posts raw
 * { latitude, longitude }; the server reverse-geocodes and stores ONLY
 * the resolved city (native_last_city). Raw coordinates are never
 * persisted or returned
 */
export default (plugin: any) => {
  plugin.controllers.user.updateMyLocation = async (ctx: any) => {
    try {
      const user = ctx.state.user;
      if (!user) return ctx.unauthorized('You must be logged in to update location.');

      const { latitude, longitude } = ctx.request.body || {};
      if (typeof latitude !== 'number' || typeof longitude !== 'number') {
        return ctx.badRequest('latitude and longitude (numbers) are required.');
      }

      const apiKey = process.env.GOOGLE_GEOCODING_API_KEY;
      if (!apiKey) {
        strapi.log.error('GOOGLE_GEOCODING_API_KEY is not configured.');
        return ctx.internalServerError('Location service is not configured.');
      }

      let resolvedCity: string | null = null;
      try {
        const url = `https://maps.googleapis.com/maps/api/geocode/json?latlng=${latitude},${longitude}&result_type=locality&key=${apiKey}`;
        const response = await fetch(url);
        const geo: any = await response.json();
        const localityComponent = geo?.results?.[0]?.address_components?.find((c: any) =>
          c.types.includes('locality')
        );
        resolvedCity = localityComponent ? localityComponent.long_name : null;
      } catch (err) {
        strapi.log.error('Reverse geocoding failed', err);
        return ctx.internalServerError('Could not resolve location.');
      }

      if (!resolvedCity) return ctx.badRequest('Could not resolve a city for the given coordinates.');

      const updated: any = await strapi.db.query('plugin::users-permissions.user').update({
        where: { id: user.id },
        data: { native_last_city: resolvedCity },
      });

      ctx.send({ success: true, native_last_city: updated.native_last_city });
    } catch (error) {
      strapi.log.error('Error updating user location:', error);
      ctx.internalServerError('Failed to update user location');
    }
  };

  plugin.routes['content-api'].routes.push({
    method: 'POST',
    path: '/users/me/location',
    handler: 'user.updateMyLocation',
    config: { policies: [], middlewares: [], prefix: '' },
  });

  return plugin;
};
